Supported SASL Mechanisms

Supported SASL Mechanisms

  • ANONYMOUS: Explicitly request the anonymous identity
  • PLAIN: Sends username and password across the wire in the clear, just like JAAS (Not recommended!)
  • EXTERNAL: Use the authenticated identity of the outer security layer (e.g. TLS client cert)
  • DIGEST-MD5: Protects username/password, but is obsolete
  • SCRAM-SHA-*: Protects username/password with a strong algorithm
